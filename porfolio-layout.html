Item Management 2.0

Revamp the current Item Management System to cater to the needs of each department.

Project Overview

The Problem:

The current item management system lacks a comprehensive solution, making it difficult for departments to efficiently retrieve relevant data, highlighting the need for a redesigned system to provide centralized data access.

Redesigning the Item Management System is necessary to create a centralized repository with an intuitive interface, robust search function, and seamless integration to enable data easy access.

Challenges:

  • The current UI is simple and on a single page.
  • Item's creating and updating are handled by 1 department.
  • The rest departments only can use the data and calculate by themselves.

Goals:

  • Customization: Tailor the system to meet the specific requirements of each department.
  • Keep it concise: Simplify the new system to one page.
  • Collaboration: Involve all departments in the process of creating and updating the data.
  • Search function: Enhance the search function to meet varying needs.

My Role:

Product Designer (Research, Information Architecture, User Flow, Wireframe, Prototype, UI)

Discover

research and findings

When an organization has multiple departments, it is inevitable that the diverse needs of each department will present unique challenges. To effectively address these challenges, it is important to adopt a structured and methodical approach.
